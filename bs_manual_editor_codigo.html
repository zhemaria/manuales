📚 Manual del Editor de Código de Brightspace

Guía completa para desarrolladores y educadores - Prof. José M. Fernández

🎯 Introducción

Esta guía recopila todas las características, trucos y mejores prácticas para trabajar con el editor de código de Brightspace. Cada sección está diseñada para ser accesible y práctica, siguiendo los principios del Diseño Universal para el Aprendizaje (DUA).

🎨 Jerarquía de Estilos CSS en Brightspace

Niveles de Control de Estilos (de Mayor a Menor Prioridad)

Nivel 1: Estilos en Línea

Los estilos más específicos aplicados directamente a elementos individuales.

  • Acceso: Seleccionar elemento → Clic en ícono + → "Attributes" → Campo "Style"
  • Ejemplo: margin-left: 50px;
  • Ventaja: Cambios inmediatos y específicos
  • Desventaja: Difícil mantenimiento, no escalable
Precaución: Evitar el uso excesivo. Cada cambio futuro requiere edición manual página por página.

Nivel 2: Etiqueta <style> en Página

Estilos que afectan a toda una página HTML específica.

<style>
    .courseware-video-wrapper {
        background-color: transparent !important;
        border: none;
    }
</style>
  • Acceso: Editor de código fuente (</>)
  • Ubicación: Dentro del <head> del documento
  • Uso ideal: Anulaciones específicas de una sola página

Nivel 3: Archivo custom.css (Plantilla)

El método más potente y recomendado para estilos a nivel de curso.

  • Ubicación: Dentro del paquete de plantilla HTML (.zip)
  • Alcance: Todo el curso que use la plantilla
  • Ventaja: Consistencia total, fácil mantenimiento
  • Implementación: Subir plantilla → Descomprimir → Activar en Settings

Nivel 4: Variables de Configuración Organizacional

Control administrativo a nivel institucional.

  • Variable clave: d2l.Tools.ContentStyler.CssOverride
  • Controlado por: Administradores de plataforma
  • Alcance: Toda la unidad organizativa
  • Prioridad: Puede anular personalizaciones de curso
Consejo de Depuración: Si un estilo no se aplica como esperabas, verifica la jerarquía completa. Un estilo de nivel superior podría estar anulando tu código.

📦 Sistema de Plantillas HTML

Implementación de Plantillas

Proceso de Instalación

  1. Descargar archivo .zip de la plantilla
  2. Ir a Manage Files en el curso
  3. Cargar el archivo .zip
  4. Descomprimir directamente en Brightspace
  5. Navegar a Content → Settings
  6. Activar checkbox: "Enable HTML Templates"

Versiones de Plantillas: V3 vs V5

Plantillas V3 (Legacy) DEPRECADO

  • Base tecnológica: Bootstrap + jQuery
  • Ventaja: Conocimiento transferible de Bootstrap
  • Clases útiles de Bootstrap:
    • rounded-lg - Esquinas redondeadas
    • rounded-circle - Imagen circular
    • img-thumbnail - Borde con padding
    • embed-responsive embed-responsive-16by9 - Videos responsivos
<!-- Video responsivo V3 -->
<div class="embed-responsive embed-responsive-16by9">
    <iframe class="embed-responsive-item" src="..."></iframe>
</div>

Plantillas V5 (Modernas) RECOMENDADO

  • Base tecnológica: Framework propietario D2L
  • Integración: Creator+ friendly
  • Estructura de archivos: Requiere ruta /shared/ en Public Files
  • Ventajas:
    • Optimización específica para Brightspace
    • Mejor integración con Content Styler
    • Actualizaciones automáticas de D2L
Error común V5: Si los estilos aparecen "rotos", verifica que la ruta a /shared/ sea correcta en los archivos CSS/JS.

Estructura del custom.css

/* Variables CSS personalizadas */
:root {
    --primary-color: #003366;
    --secondary-color: #0066cc;
    --font-family-main: 'Arial', sans-serif;
    --spacing-unit: 1rem;
}

/* Anulaciones globales */
body {
    font-family: var(--font-family-main);
    line-height: 1.6;
}

/* Componentes específicos */
.d2l-htmlblock h1 {
    color: var(--primary-color);
    border-bottom: 2px solid var(--secondary-color);
}

/* Responsividad */
@media (max-width: 768px) {
    .custom-container {
        padding: var(--spacing-unit);
    }
}
Mejores prácticas: Usa variables CSS para mantener consistencia y facilitar cambios globales de tema.

♿ Accesibilidad y Estándares WCAG

Principios Fundamentales de Accesibilidad

1. Estructura Semántica

  • Jerarquía de Encabezados:
    • Un solo H1 por página (título principal)
    • No saltar niveles (H1 → H3 ❌)
    • Orden lógico: H1 → H2 → H3 → H4
  • Listas Correctas:
    • Usar herramientas del editor, no guiones manuales
    • UL para listas no ordenadas
    • OL para listas numeradas

2. Contraste de Color

Tipo de Contenido Ratio Mínimo Nivel WCAG
Texto normal 4.5:1 AA
Texto grande (18pt+) 3:1 AA
Texto normal (mejorado) 7:1 AAA
Texto grande (mejorado) 4.5:1 AAA

3. Enlaces Descriptivos

Evitar:
  • "Haga clic aquí"
  • "Más información"
  • "Ver más"
  • URLs sin contexto
Usar en su lugar:
  • "Descarga el programa del curso (PDF)"
  • "Visita la biblioteca universitaria"
  • "Lee el artículo sobre metodología"

4. Texto Alternativo para Imágenes

<!-- Imagen informativa -->
<img src="grafico.jpg" 
     alt="Gráfico mostrando aumento del 30% en matriculaciones 2020-2024">

<!-- Imagen decorativa -->
<img src="decoracion.jpg" alt="">

<!-- Imagen compleja -->
<img src="diagrama.jpg" 
     alt="Diagrama del ciclo del agua" 
     longdesc="#descripcion-detallada">

5. Formularios Accesibles

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre" required>

<fieldset>
    <legend>Selecciona tu programa:</legend>
    <input type="radio" id="grado" name="programa" value="grado">
    <label for="grado">Grado</label>
    <input type="radio" id="master" name="programa" value="master">
    <label for="master">Máster</label>
</fieldset>

Herramientas de Validación

  • Brightspace Accessibility Checker: Integrado en el editor
  • WAVE: Extensión del navegador
  • axe DevTools: Para desarrolladores
  • Contrast Ratio Checker: Para validar colores
Recordatorio: La accesibilidad no es opcional. Es un requisito legal en muchas jurisdicciones y fundamental para la inclusión educativa.
🔍 Solución de Problemas Comunes

Problemas Frecuentes y Soluciones

1. Los estilos CSS no se aplican

  • Causa: Conflicto con estilos de mayor prioridad
  • Solución:
    • Verificar la jerarquía de estilos
    • Usar !important con moderación
    • Aumentar la especificidad del selector
/* Aumentar especificidad */
.d2l-htmlblock .mi-clase { /* Más específico */
    color: blue;
}

/* Último recurso */
.mi-clase {
    color: blue !important;
}

2. Plantilla V5 aparece "rota"

  • Causa: Ruta incorrecta a archivos CSS/JS
  • Solución:
    1. Verificar que existe la carpeta /shared/
    2. Comprobar rutas en el HTML de la plantilla
    3. Asegurarse de que los archivos están en Public Files

3. Videos no responsivos

<!-- Solución para V3 -->
<div class="embed-responsive embed-responsive-16by9">
    <iframe class="embed-responsive-item" src="..."></iframe>
</div>

<!-- Solución universal -->
<style>
.video-container {
    position: relative;
    padding-bottom: 56.25%; /* 16:9 */
    height: 0;
    overflow: hidden;
}
.video-container iframe {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
}
</style>

4. Imágenes muy grandes

/* CSS para controlar tamaño máximo */
.d2l-htmlblock img {
    max-width: 100%;
    height: auto;
}

/* Tamaño específico manteniendo proporción */
.imagen-mediana {
    width: 500px;
    max-width: 100%;
    height: auto;
}

5. Acordeón no se abre/cierra

  • Verificar: JavaScript de la plantilla cargado
  • Comprobar: IDs únicos en cada panel
  • Solución CSS pura:
<details>
    <summary>Título del acordeón</summary>
    <p>Contenido que se expande/colapsa</p>
</details>

6. Problemas de caché

Forzar recarga sin caché:
  • Windows/Linux: Ctrl + Shift + R
  • Mac: Cmd + Shift + R
  • Añadir versión a archivos: custom.css?v=2
🔄 Migración y Compatibilidad de Versiones

Estrategias de Migración V3 → V5

Enfoque Recomendado: Migración Gradual

  1. Evaluación inicial:
    • Inventariar páginas con V3
    • Identificar componentes críticos de Bootstrap/jQuery
    • Documentar personalizaciones existentes
  2. Estrategia dual:
    • Mantener contenido V3 existente funcionando
    • Crear todo contenido nuevo en V5
    • Migrar páginas críticas primero
  3. Conversión de componentes:
Componente V3 Equivalente V5 Notas de Migración
.container-fluid .d2l-container Cambiar clase en HTML
.row / .col-md-* Grid Creator+ Usar herramienta visual
.btn-primary .d2l-button-primary Actualizar clases
.alert-info Callout Component Recrear con Creator+
jQuery animations CSS transitions Reescribir en CSS puro
